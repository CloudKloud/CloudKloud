Type 1. Instance Compromise

ITEM Severity
쉘 명령어 사용 내역 (yum)
패키지 매니저를 사용하여 패키지를 설치/수정/삭제 할 수 있다
Low
쉘 명령어 사용 내역 (sudo)
루트 권한을 사용하여 명령어를 수행하기에, 시스템 명령 및 권한이 필요한 것을 수행할 수 있다
Low
쉘 명령어 사용 내역 (service)
해당 명령을 통해 시스템 및 유저 서비스를 시작/종료할 수 있다
Medium
쉘 명령어 사용 내역 (cron)
해당 명령어를 통해 시스템에서 주기적으로 실행할 수 있는 명령 및 프로그램을 지정 가능하여, 이를 통해 시스템에 위해를 가할 수 있다
Medium
인스턴스 생성, 실행
특정 인스턴스 생성 및 실행은 과금 및 계정 전반에 영향을 미칠 수 있는 사항이다
Low
인스턴스 중지, 삭제
특정 인스턴스가 중지/삭제되는 것은 사용자가 서비스 및 데이터 운용 시 큰 손해가 일어날 수 있다
Medium
보안그룹 수정
보안그룹 수정을 통해 네트워크 접근 권한 등을 변조할 수 있음
Medium
침투 테스트 시스템에서의 API 호출
침투 테스트 도구를 사용하여 AWS 환경에 대한 무단 액세스 권한을 얻기도 한다
Medium

Type 2. IAM Compromise

ITEM Severity
침투 테스트 시스템에서의 API 호출
공격자들은 침투 테스트 도구를 사용하여 AWS 환경에 대한 무단 액세스 권한을 얻곤 한다
Medium
네트워크 액세스 권한 변경
공격자가 공격을 위해 다양한 포트에서 특정 인바운드 트래픽을 허용하도록 보안 그룹을 변경하여 EC2 인스턴스에 대한 액세스 능력을 개선하도록 시도하는 경우가 많다
Medium
Cloudtrail 로깅 중단 및 기존 로그 삭제
공격자가 추후 공격 수행이 들키지 않게 로깅 비활성화 및 기존 로그를 삭제했을 가능성이 있다
Medium
IAM 사용자, 그룹, 정책 등의 수정 및 삭제
액세스 포인트가 닫혀 있어도 공격자는 도난된 자격 증명을 사용하여 사용자 생성, 그룹 정책 추가, 권한 변경 등의 행위를 할 가능성이 있음
Medium
리소스 보안 액세스 정책 변경
자격 증명이 도난된 경우 공격자가 리소스 정책을 변경하여 해당 리소스에 대한 액세스 권한을 얻을 수 있다
Medium
Root 계정으로 API 호출
AWS 루트 계정은 AWS의 모든 리소스에 제한없이 접근이 가능하며 사용 권한을 제한할 방법이 존재하지 않는다. 그러므로 루트 계정에 대한 액세스 키를 생성하지 않는 것이 좋으며, 루트 계정의 사용을 자제하여야 한다
Low
컴퓨팅 리소스 시작
공격자가 도난된 자격 증명을 사용하여 컴퓨팅 리소스를 시작해 채굴 또는 암호 크래킹을 할 가능성이 있고, 또한 공격자가 EC2 인스턴스와 그 자격증명을 사용하여 계정 액세스를 유지하는 신호일 수 있다
Low

Type 3. S3 Compromise

ITEM Severity
버킷 목록 검색
공격자가 정보를 수집하여 AWS 환경이 더 광범위한 공격에 취약해졌는지 확인하는 공격의 검색 단계와 관련되어 있다
Low
버킷 내 데이터 생성
공격자가 공격을 위한 파일을 업로드했을 가능성이 있다
Low
버킷 내 데이터 삭제
공격자가 중요한 데이터를 삭제했을 가능성이 있다
Medium
권한이 없는 IAM 개체의 S3 API 호출
이미 공격자의 공격이 성공해 데이터 유출 단계일 가능성이 있다
Medium
서버 액세스 로깅 비활성화
공격자가 추후 공격 수행이 들키지 않게 로깅을 비활성화했을 가능성이 있다
Medium
버킷 또는 객체의 권한 변경
공격자가 계정 외부에서 정보를 공유할 수 있도록 하기 위해 수행했을 가능성이 있다
Medium
버킷 정책 변경
버킷을 권한이 없는 사용자들에게 공개적으로 액세스 가능하게 할 수 있다
Medium
침투 테스트 시스템에서 API 호출(KALI, PARROT, PENTOO...)
침투 테스트 도구를 사용하여 AWS 환경에 대한 무단 액세스 권한을 얻기도 한다
Medium

Type 4. DB Compromise

ITEM Severity
DB 목록 검색
공격자가 정보를 수집하여 AWS 환경이 더 광범위한 공격에 취약해졌는지 확인하는 공격의 검색 단계와 관련되어 있다
Low
DB 내 데이터 삭제
공격자가 중요한 데이터를 삭제했을 가능성이 있다
Medium
DB 사용자 추가
공격자가 많은 권한을 가진 사용자를 추가하여 모든 DB에 접근하여 정보 유출을 할 가능성이 있다
Medium
DB 사용자 권한 변경
공격자가 DB 사용자의 권한을 임의로 변경하여 모든 DB에 접근하여 정보 유출을 할 가능성이 있다
Medium
권한이 없는 IAM 개체의 RDS API 호출
이미 공격자가 공격하여 데이터 유출 단계일 가능성이 있다
Medium
RDS 로깅 비활성화 (Parameter 그룹 수정)
공격자가 추후 공격 수행이 들키지 않게 로깅을 비활성화했을 가능성이 있다
Medium
RDS 로깅 비활성화 (Parameter 그룹 삭제)
공격자가 추후 공격 수행이 들키지 않게 로깅을 비활성화했을 가능성이 있다
Medium
DB 작동 중지
공격자가 DB를 중지시켜 DB 내 데이터를 사용하지 못하게 할 수 있다
Medium
DB 삭제
공격자가 중요한 데이터를 삭제했을 가능성이 있다
Medium
침투 테스트 시스템에서의 API 호출
침투 테스트 도구를 사용하여 AWS 환경에 대한 무단 액세스 권한을 얻기도 한다
Medium
RDS 로깅 비활성화 (Cloudwatch 로깅 수정)
공격자가 추후 공격 수행이 들키지 않게 로깅 비활성화 및 로깅 수정했을 가능성이 있다
Medium
DB 연결 실패
권한이 없는 사용자가 접근하여 연결 실패됐을 경우, 사용자가 공격자일 가능성이 있다
Medium
